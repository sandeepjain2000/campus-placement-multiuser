1 · Data Tables & CRUD Completeness 4 issues

Data tables (like those on the Documents page) lack advanced SaaS features such as sorting, pagination, search, and export. CRUD operations are missing soft-delete capability.
Category Screen / Component Issue Severity Recommended Fix
Data Tables Document Table
documents/page.js		 Missing pagination, search, and sorting. As the dataset scales, rendering all records will degrade performance and usability. High Implement server-side pagination and sorting. Add a debounced search input to filter documents by name or type.
CRUD: Delete All Delete Actions Hard deletes are used instead of soft deletes. There is no "undo" or recovery mechanism once a record is removed. Medium Implement a deleted_at timestamp for soft deletes. Provide a toast notification with an "Undo" action valid for 5-10 seconds.
Bulk Actions List Views No ability to select multiple rows for batch operations (e.g., bulk delete, bulk status update). Medium Add a checkbox column to data tables to enable bulk actions. Display a floating action bar when items are selected.
Export & Reporting Dashboard Views Missing export functionality. Users cannot download lists (CSV/Excel) or generate PDF reports for offline use. Low Add export buttons to data tables. Ensure exports respect the current filters and sorting applied to the view.

2 · State, Edge Cases & Performance 4 issues

The application lacks robust handling for network failures, background refreshes, and offline states. Infinite loading skeletons erode trust during API errors.
Category Screen / Component Issue Severity Recommended Fix
API Failures All SWR Hooks Unhandled SWR error states result in infinite loading skeletons when network requests fail. High Create a global <PageError onRetry={mutate} /> component and render it whenever SWR returns an error.
Offline Indicator Global Layout No offline indicator or recovery mechanism. Users are not informed if their internet connection drops while using the app. High Listen to window.addEventListener('offline') and show a persistent offline banner. Re-sync pending actions on 'online'.
Background Sync Dashboard Pages Stale data may persist if a user returns to a tab after a long period. No explicit refresh prompts. Medium Enable SWR revalidateOnFocus and consider showing a subtle "New data available" toast for large updates.
Empty States Dashboard Widgets Hardcoded placeholder data is displayed (e.g., "↑ 2 this week") instead of true empty/zero-state guidance. Medium Implement explicit empty states that provide clear guidance and primary CTAs (e.g., "Upload your first document") instead of fake data.

3 · Accessibility & Navigation 5 issues

Accessibility fails due to reliance on emojis for iconography, missing ARIA attributes, and lack of keyboard navigation aids like skip links.
Category Screen / Component Issue Severity Recommended Fix
Iconography Navigation & Menus Extensive use of emojis as icons. Screen readers announce emojis verbatim, creating a confusing navigational experience. High Replace all emojis with accessible SVG icons (e.g., Lucide React) with aria-hidden="true".
Keyboard Navigation Global Layout No "Skip to Main Content" link. Keyboard users must tab through the entire sidebar to reach the main content area. High Add a visually hidden skip link that becomes visible on focus at the start of the DOM.
State Indication Sidebar Menu Active menu items rely solely on visual CSS classes and lack aria-current="page" to inform screen readers. High Dynamically add aria-current="page" to the active navigation link.
Live Regions Forms & Actions Success and error messages are not wrapped in ARIA live regions, so they are not announced to screen reader users. Medium Add role="alert" or role="status" to form error banners and toast notifications.
Navigation Context List Views Scroll position and filter states are lost when navigating away from and back to a list view. Low Store list filter states in URL query parameters or session storage to preserve context across navigation.

4 · Interaction & Feedback Loops 4 issues

Micro-interactions lack polish. Reduced motion preferences are ignored, focus indicators are missing, and action feedback latency is not optimized.
Category Screen / Component Issue Severity Recommended Fix
Focus Visibility Global Styles Buttons lack a visible :focus-visible outline, making keyboard navigation nearly impossible to track. High Define a clear, high-contrast outline and outline-offset for :focus-visible states in globals.css.
Reduced Motion Animations The UI does not respect the OS-level prefers-reduced-motion setting, forcing animations on all users. Medium Add a global CSS media query to disable or drastically reduce animations when reduced motion is preferred.
Action Feedback Buttons Loading buttons collapse their width due to changing text, and lack proper spinner feedback. Medium Maintain button min-width during loading states and insert an animated spinner alongside the loading text.
Hover Intent Dropdowns / Menus Hover-triggered elements (like notifications) open instantly, which can lead to accidental activation when moving the mouse. Low Introduce a slight hover delay (100-150ms) for mega-menus and tooltips to confirm user intent.

5 · Security UX & Compliance 3 issues

Security UX is compromised by plaintext password fields and explicit error messages. Compliance features like data export and account deletion are missing.
Category Screen / Component Issue Severity Recommended Fix
Credential Exposure Login Form Password field uses type="text", making passwords visible and preventing password managers from working correctly. High Change to type="password" immediately. Add a toggle (eye icon) to allow users to view the password if needed.
User Enumeration Login Errors Detailed authentication errors are passed directly to the UI, potentially allowing attackers to verify registered email addresses. Medium Display a generic "Invalid email or password" error message to prevent user enumeration attacks.
Compliance User Settings Missing "Export my data" and "Delete my account" workflows, which are essential for GDPR/CCPA compliance. Medium Implement secure self-service workflows for data export and account deletion with appropriate re-authentication steps.

6 · Multi-Tenancy, Roles & Audit 3 issues

Role definitions and menus are hardcoded rather than database-driven. There is a lack of audit logging for administrative actions.
Category Screen / Component Issue Severity Recommended Fix
Hardcoded RBAC Menus & Routes Dashboard menus and valid roles are hardcoded in configuration files (e.g., dashboardMenu.js, validators.js). High Migrate role definitions and menu structures to the database to support dynamic permissions and future role expansions.
Audit Logs Admin Actions No audit trail exists for critical operations (e.g., approving drives, updating statuses) performed by College Admins or Employers. Medium Implement a secure audit logging table to track "who did what and when" for all destructive or status-altering actions.
Simultaneous Edits Shared Resources No detection or locking mechanism when two admins edit the same resource (e.g., a Placement Drive) simultaneously. Low Implement optimistic concurrency control using a version column, or add presence awareness/locking for shared forms.
